Have a look at the network connections in the Browser debug console.
  • If you update the parent, the child is rendered as an empty div.
    A single request is made for the parent. This is request executed by the change of the input in the parent component. No other request is executed. This is probably because the data-live-id doesn't change. But what about the dataModel and animal arguments of child component ?
    Bug

  • If you update the child, the parent is updated, but the child is rendered as an empty div.
    A single request is made for the parent, without waiting for the debounce delay. I guess this is the request based on dataModel argument of child component.
    Bug

  • If you then update the parent or the child, the child is rendered and updated.
    A single request is made for the parent.
    Semi Bug. By the way, at the next update, the child component will not be rendered correctly.
